# Non-Ergodic Mess3: Belief State Geometry

[![Open in Colab](https://colab.research.google.com/assets/colab-badge.svg)](https://colab.research.google.com/github/elenaajayi/mess3-belief-geometry/blob/main/MATS_ELENA_SIMPLEX.ipynb)
[Read the final report](paper/report.pdf)

This repository studies how a small transformer represents belief states when its training data comes from a non-ergodic mixture rather than one stationary process. Each length-16 sequence is generated by one of three Mess3 hidden Markov model components (`alpha = 0.4, 0.7, 0.9`), but the model is not told which component produced it. A three-layer transformer is trained for next-token prediction, then its residual-stream activations are analyzed with PCA, linear probes, activation patching, layer decomposition, cross-component transfer, and out-of-distribution tests. The five pre-registered predictions produced two confirmed results, one partial result, and two refutations. The central finding is that belief states are recoverable from a low-dimensional shared subspace, while the three components do not form separate geometric clusters.

## Reported results

| Prediction | Evidence reported in the paper | Outcome |
| --- | --- | --- |
| 1. Components share a belief-state subspace | Five dimensions explain 96% of the variance; subspace overlap is `0.96–0.99`. | Confirmed |
| 2. Belief tracking and component identification improve with depth | Belief-state probe R² rises from `0.618` at the embedding to `0.842` at the final layer, but silhouette scores remain negative. | Partial |
| 3. Components gradually separate with more context | Silhouette remains approximately `-0.03` across context positions. | Refuted |
| 4. Component separation appears sharply during training | No sharp transition appears; belief R² rises smoothly from about `0.74` to `0.85`. | Refuted |
| 5. The final layer is sufficient for belief recovery | Final-layer R² is `0.849`, compared with `0.853` for all layers combined (`0.004` gap). | Confirmed |

**Summary:** 2 of 5 predictions confirmed, 1 partially confirmed, and 2 refuted.

These values are transcribed from the [final report](paper/report.pdf) and the notebook's retained outputs. The repository cleanup did not rerun the full 15,000-step training experiment.

## Run the experiment

### Google Colab

Open the notebook using the Colab badge above, select a GPU runtime, and run the cells in order. The notebook attempts to mount Google Drive and saves figures and checkpoints under `MyDrive/mess3_belief_geometry/`.

### Local JupyterLab

Python 3.10 or newer is required. A CUDA-capable GPU is recommended for the full training run, but the notebook falls back to CPU.

```bash
python -m venv .venv
source .venv/bin/activate
python -m pip install -r requirements.txt
jupyter lab MATS_ELENA_SIMPLEX.ipynb
```

When run outside Colab, the notebook writes plots to `figures/` and model checkpoints to `checkpoints/`.
